# 리눅스 과제 연습하기


## 1. top (Table of Processes)
시스템 리소스 사용량과 실행 중인 프로세스를 **실시간** 으로 모니터링하는 대화형 도구입니다.

* **주요 기능**: CPU/메모리 점유율 실시간 확인, 시스템 부하(Load Average) 체크
* **자주 쓰는 옵션**:
  * `top -d [초]`: 화면 갱신 주기 설정 (예: `top -d 1`)
  * `top -p [PID]`: 특정 프로세스 ID만 지정해서 모니터링
* **실행 중 주요 단축키**:
  * `M`: 메모리 사용률이 높은 순 정렬
  * `P`: CPU 사용률이 높은 순 정렬
  * `q`: 프로그램 종료

---

## 2. ps (Process Status)
현재 실행 중인 프로세스의 **정적인 스냅샷** 을 출력합니다. (명령어를 실행한 그 순간의 상태 조회)

* **주요 기능**: 특정 프로그램의 실행 여부 확인 및 PID 확인
* **자주 쓰는 옵션 조합**:
  * `ps -ef`: 시스템의 모든 프로세스를 상세한 정보(UID, PID, PPID 등) 와 함께 출력
  * `ps aux`: CPU 및 메모리 사용률을 포함하여 시스템 내 모든 프로세스 출력

---

## 3. jobs
현재 터미널 쉘 세션 안에서 **백그라운드로 실행 중이거나 정지된 작업(Job)** 들의 목록을 보여줍니다.

* **주요 특징**: 시스템 전체 프로세스가 아니라, 내가 현재 사용 중인 터미널 창 내에서 발생한 작업만 관리합니다.
* **출력 상태**:
  * `Running`: 백그라운드에서 정상 실행 중인 상태
  * `Stopped`: `Ctrl + Z` 를 눌러 실행이 일시 정지된 상태
* **자주 쓰는 옵션**:
  * `jobs -l`: 작업 번호와 함께 해당 프로세스의 고유 PID 까지 상세히 출력합니다.

---

## 4. kill
실행 중인 프로세스나 작업에 **시그널(Signal) 을 보내 제어** 하는 명령어입니다. (주로 종료 요청 시 사용)

* **핵심 시그널 종류**:
  * `15 (SIGTERM)`: **기본 시그널**. 프로세스에게 정상적인 종료를 요청하며, 하던 작업을 안전하게 정리하고 닫히도록 합니다.
  * `9 (SIGKILL)`: **강제 종료**. 프로세스가 신호를 무시할 수 없으며 커널 수준에서 즉시 소멸시킵니다. (데이터 유실 위험 있음)
* **사용 예시**:
  * **PID 기준 강제 종료**: 
    `kill -9 1234`
    (PID 가 1234인 프로세스를 강제 종료)
  * **작업 번호 기준 정상 종료**: 
    `kill -15 %1`
    (`jobs` 명령어에서 확인한 대괄호 속 1번 백그라운드 작업을 정상 종료)
